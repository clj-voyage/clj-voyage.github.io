my-zero?

Clojure has function zero?, which accepts one number and returns true, if that number is equal to zero, and false otherwise. Write function my-zero?, which does the same. Don't use built-in zero?. If you know if, don't use it.

Tests with the expected results: 

(my-zero? 0)
=> true
(my-zero? 0.0)
=> true
(my-zero? 1)
=> false
(defn my-zero? [x]
  (== x 0))

my-pos?, my-neg?

Clojure has function pos?, which accepts one number and returns true, if that number is positive, and false otherwise. Write function my-pos?, which does the same. Don't use built-in pos?. If you know if, don't use it.

There is also function neg?, which accepts one number and returns true, if that number is negative, and false otherwise. Write function my-neg?, which does the same. You can use your my-pos?. Don't use built-in neg?. If you know if, don't use it.

(defn my-pos? [x]
  (> x 0))

(defn my-neg? [x]
  (> 0 x))

(defn my-neg? [x]
  (not (my-pos? x)))

my-even?, my-odd?

Clojure has function even?, which accepts one number and returns true, if that number is even, and false otherwise. Write function my-even?, which does the same.

You can use your my-zero? from previous exercise. Don't use built-in even?. If you know if, don't use it.

There is also function odd?, which accepts one number and returns true, if that number is odd, and false otherwise. Write function my-odd?, which does the same.

You can use your my-even? or my-zero?. Don't use built-in odd?. If you know if, don't use it.

(defn my-even? [x]
  (my-zero? (rem x 2)))

(defn my-odd? [x]
  (not (my-zero? (rem x 2))))

(defn my-odd? [x]
  (not (my-even? x)))
